Introduction

The Warranty Management System is a comprehensive web-based application designed to streamline the process of handling warranty claims. It provides tools for claim submission, tracking, reporting, and analysis, helping organizations efficiently manage their warranty service operations.

Key features include:

  • Centralized claim management
  • Automated workflow for claim processing
  • Comprehensive reporting and analytics
  • File attachment support for claim documentation
  • User role-based access control
  • Email notifications
  • SLA (Service Level Agreement) tracking

Getting Started

System Requirements

To access and use the Warranty Management System, you need:

  • A modern web browser (Chrome, Firefox, Safari, or Edge)
  • Internet connection
  • Valid user credentials

Login

To access the system:

  1. Navigate to the system URL: http://trapo.com/wms
  2. Enter your email address
  3. Click "Request OTP"
  4. Check your email for the one-time password (OTP)
  5. Enter the OTP in the login form
  6. Click "Verify & Login"

The OTP is valid for 10 minutes. If you don't receive the email, check your spam folder or click "Resend OTP" on the login page.

User Roles

The system has several user roles, each with different permissions:

  • Administrator: Full access to all system features and settings
  • Manager: Access to claims, reports, and limited settings
  • Agent: Access to claims assigned to them
  • Customer: Access to submit and view their own claims

Dashboard

Dashboard Overview

The dashboard provides a quick overview of the warranty claim system status, including:

  • Total number of open claims
  • Claims by status
  • Claims by category
  • Recent activity

Claims Overview

The Claims Overview section displays statistics about claims, including:

  • Claims by status (New, In Progress, Resolved, Closed)
  • Claims by category
  • Claims by priority

You can filter this data by time period (This Week, This Month, This Year) using the dropdown at the top of the section.

Claims Management

Viewing Claims

To view claims:

  1. Navigate to the Claims section from the main menu
  2. Use the filters to narrow down the list by category, status, or assigned person
  3. Click on a claim to view its details

Creating Claims

To create a new warranty claim:

  1. Click the "New Claim" button in the Claims section
  2. Fill in the required information:
    • Customer information (name, email, phone)
    • Order details (order ID, delivery date)
    • Product information (SKU, product type)
    • Issue description
  3. Upload supporting documentation (images up to 2MB, videos up to 10MB in MP4 or MOV format)
  4. If phone number or email is not present(for eg: Shopee, Lazada, Tiktok order), ask the customer and fill that.
  5. We need these to send information to the customer regarding the claims.
  6. You can also add relevant person by using ‘@‘. Do note, you can only add registered users.
  7. Click "Submit"

Note: The system prevents duplicate claims for the same order ID and SKU combination.

Updating Claims

To update an existing claim:

  1. Open the claim from the Claims list
  2. Click the "Edit" button
  3. Update the relevant information
  4. Click "Save Changes"

Assigning Claims

To assign a claim to a user:

  1. Open the claim from the Claims list
  2. Click the "Assign" button
  3. Select a user from the dropdown
  4. Click "Assign"

Reports

The system provides several reports to analyze warranty claim data. These reports help managers and administrators track claim performance, identify trends, and make data-driven decisions.

Claim Performance Report

This report shows the performance metrics for claims processing, including:

  • Average resolution time
  • Claims by status
  • Claims by agent
  • SLA compliance percentage
  • Customer satisfaction ratings

To access this report:

  1. Navigate to the Reports section from the main menu
  2. Select "Claim Performance" from the reports dropdown
  3. Set the date range for the report
  4. Click "Generate Report"

The report can be exported in CSV, PDF, or Excel format using the export buttons at the top of the report.

Screenshot: Claim Performance Report

Monthly Claims Report

This report provides a monthly breakdown of claims by category, displayed in a tabular format with:

  • Months as columns
  • Categories as rows
  • Totals for each category and month
  • Month-over-month percentage changes
  • Year-to-date totals

To access this report:

  1. Navigate to the Reports section from the main menu
  2. Select "Monthly Claims" from the reports dropdown
  3. Select the year for the report
  4. Click "Generate Report"

You can drill down into specific categories by clicking on the category name in the report. The system also provides visual charts and graphs to help visualize the data.

Screenshot: Monthly Claims Report

Custom Reports

The system allows administrators to create custom reports based on specific requirements:

  1. Navigate to the Reports section from the main menu
  2. Click on "Custom Reports"
  3. Click "New Report"
  4. Select the data fields to include
  5. Set filtering criteria
  6. Choose display options (table, chart, etc.)
  7. Save the report with a descriptive name

Custom reports can be saved and shared with other users. They can also be scheduled to run automatically and be sent via email to designated recipients.

Warranty Rules Management

The Warranty Rules Management module allows administrators to define and manage warranty policies for different product types. This ensures consistent application of warranty terms across all claims and helps maintain standardized warranty processing.

Viewing Warranty Rules

To view existing warranty rules:

  1. Navigate to the Warranty Rules section from the main menu
  2. Browse the list of existing rules
  3. Click on a rule to view its details

The rule details page shows:

  • Product type covered by the rule
  • Warranty duration
  • Coverage details
  • Exclusions
  • Required documentation for claims
  • Approval workflow

Screenshot: Warranty Rule Details

Creating Warranty Rules

To create a new warranty rule:

  1. Click the "New Rule" button in the Warranty Rules section
  2. Fill in the required information:
    • Product type
    • Warranty period (in months)
    • Coverage details (parts, labor, shipping)
    • Exclusions (misuse, natural disasters, etc.)
    • Required documentation (proof of purchase, photos)
    • Approval workflow (auto-approve, manager review)
  3. Click "Save Rule"

Note: New warranty rules will apply to all future claims for the specified product type. Existing claims will continue to use the rules that were in effect when the claim was created.

Screenshot: Creating a New Warranty Rule

Editing Warranty Rules

To edit an existing warranty rule:

  1. Navigate to the Warranty Rules section
  2. Find the rule you want to edit in the list
  3. Click the "Edit" button
  4. Make the necessary changes
  5. Click "Save Changes"

When editing a warranty rule, you can choose to apply the changes to:

  • All future claims only
  • All open claims (not yet resolved)
  • All claims (including previously resolved claims)

Caution: Applying changes to previously resolved claims may require manual review and could have legal implications.

Warranty Rule Templates

The system includes pre-defined warranty rule templates for common product types:

  • Electronics (12-month standard warranty)
  • Appliances (24-month standard warranty)
  • Furniture (36-month limited warranty)
  • Clothing (30-day quality guarantee)

To use a template:

  1. Click "New Rule" in the Warranty Rules section
  2. Select "Use Template" from the dropdown
  3. Choose the appropriate template
  4. Customize the template as needed
  5. Click "Save Rule"

User Management

The User Management section allows administrators to manage system users and their permissions.

Adding Users

To add a new user to the system:

  1. Navigate to the User Management section
  2. Click "Add User"
  3. Enter the user's information:
    • Name
    • Email address
    • Role (Administrator, Manager, Agent, or Customer)
  4. Click "Save"

The system will send an email to the new user with instructions to set up their account.

Managing User Permissions

To modify a user's permissions:

  1. Find the user in the User Management section
  2. Click "Edit"
  3. Modify their role or specific permissions
  4. Click "Save Changes"

System Settings

The Settings section allows administrators to configure system-wide settings.

Email Notifications

To configure email notifications:

  1. Navigate to the Settings section
  2. Select the "Email Notifications" tab
  3. Enable or disable specific notification types
  4. Customize email templates if needed
  5. Click "Save Changes"

SLA Configuration

To configure Service Level Agreements:

  1. Navigate to the Settings section
  2. Select the "SLA Configuration" tab
  3. Set the default resolution time for different claim types
  4. Configure escalation rules
  5. Click "Save Changes"

Troubleshooting

Common issues and their solutions:

Login Issues

  • Problem: Cannot log in with correct credentials
    Solution: Ensure caps lock is off, reset your password, or contact the administrator
  • Problem: Session timeout
    Solution: Log in again. The system automatically logs out after 30 minutes of inactivity

File Upload Issues

  • Problem: Cannot upload files
    Solution: Ensure the file is under the size limit (2MB for images, 10MB for videos) and in a supported format

Common Issues

  • If you’ve tested as per the guide and still facing issues, reach out to IT department

Support Contact

For additional support, contact the IT department.

Appendix

Glossary of Terms

Claim
A formal request for warranty service or replacement of a defective product.
OTP (One-Time Password)
A temporary password sent to your email for secure authentication.
SLA (Service Level Agreement)
Deadline for a claim to be resolved
Warranty Period
The timeframe during which a product is covered by warranty protection.
Claim Status
The current state of a warranty claim (e.g., Submitted, In Review, Approved, Rejected).
SKU
Stock Keeping Unit - a unique identifier for each product.
